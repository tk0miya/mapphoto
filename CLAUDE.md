# CLAUDE.md

## プロジェクトの目的

Google Maps My Maps でエクスポートした KMZ ファイルと写真を組み合わせ、「どこで撮った写真か」を視覚化した PNG 画像を生成するツール。

Passage（iOS アプリ）が旅行写真にルートを重ねて表示するスタイルにインスパイアされ、それの道路旅行版として作成した。写真を主役に置き、地図・ルート・テキスト情報をオーバーレイとして重ねる構成にしている。

KMZ は Google Maps My Maps から個別レイヤーを「KMZ としてエクスポート」した形式（インライン KML）を想定している。マップ全体をエクスポートすると NetworkLink 形式になりブラウザから読めないため注意。

## 想定する使い方

1. Google Maps My Maps でルートを作成し、レイヤー単位で KMZ にエクスポートする
2. 旅先で撮影した写真（HEIC・JPEG）と KMZ をブラウザにドロップする
3. タイトル・サブタイトルを任意で入力する
4. 日本地図 + ルート + 撮影地点マーカー + テキスト情報が合成された PNG をダウンロードする

写真に GPS が埋め込まれていれば、撮影地点が地図上にマーカーで表示され、日時・座標もテキストとして自動で入る。

## 設計上の判断

**サーバーレス・無料運用を優先**
すべての処理をブラウザ内で完結させ、ホスティングコストをゼロにする。GitHub Pages や Netlify への静的ファイル配信を想定している。

**依存を最小限に**
Vanilla TypeScript + Vite のみで構成し、UI フレームワークは使用しない。ランタイム依存は jszip（KMZ 展開）・exifr（EXIF 読み取り）・libheif-js（HEIC デコード）の 3 つに絞っている。

**HEIC 対応**
iPhone で撮影した写真をそのまま扱えることを重視した。Safari や macOS Chrome のネイティブ対応を優先し、失敗時に libheif-js（HEVC 対応）へフォールバックする。
